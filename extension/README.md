## GreenNova Chrome Extension

GreenNova is a Manifest V3 Chrome extension that shows a local AI-powered sustainability score next to products on supported e-commerce pages.

Current implementation targets:
- Amazon
- Flipkart

When a product score is clicked, GreenNova opens a floating side panel on the current webpage with an environmental impact report generated by local Ollama using `gemma3:12b`.

## Features Implemented

- Product detection on supported domains
- Sustainability score badge (`0-100` + `A-E`) beside detected products
- Click-to-open floating report panel on page
- Local inference through Ollama (`http://localhost:11434`)
- Silent fallback when Ollama is unavailable (no badge shown)
- Basic settings page for domain allowlist

## Prerequisites

1. Install Ollama: https://ollama.com/
2. Pull the model:

```powershell
ollama pull gemma3:12b
```

3. Start Ollama service (if not auto-running):

```powershell
ollama serve
```

## Load Extension in Chrome

1. Open `chrome://extensions`
2. Enable `Developer mode`
3. Click `Load unpacked`
4. Select the `extension/` folder

## How It Works

1. `src/content.js` scans supported e-commerce pages for product cards.
2. Visible products are sent to `src/background.js` for scoring.
3. `src/background.js` calls local Ollama (`gemma3:12b`) with a strict JSON prompt.
4. Content script injects score badges next to each product.
5. Clicking a badge opens a floating panel with report details:
	 - Summary
	 - How product affects environment
	 - Positive impacts
	 - Negative impacts
	 - Recommendations

## Notes

- This version uses selector heuristics and may need updates if target sites change their DOM.
- Scoring is immediate per visible product.
- Cache is memory-based in the service worker for repeated products.

## Troubleshooting: Ollama 403 / CORS

If you see `ollama-http-403` or `OLLAMA_FORBIDDEN`, Ollama is rejecting the extension origin.

On Windows, do this in PowerShell:

```powershell
taskkill /IM ollama.exe /F
$env:OLLAMA_ORIGINS="chrome-extension://*,http://localhost,http://127.0.0.1"
ollama serve
```

Important:
- Run all 3 commands in the same terminal session.
- After restarting Ollama, reload the extension in `chrome://extensions` and refresh the e-commerce tab.
